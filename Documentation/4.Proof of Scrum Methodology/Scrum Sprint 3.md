# Proof of Scrum Methodology for Sprint 3

## Product Backlog

### User Stories


| Identifier |  User Story |
| ---------- | ------------- |
|US 1    | As a user, I should be able to create an account so that I can access funding opportunities. |
|US 2    | As a user, I should be able to submit funding applications for educational, business, or event initiatives. |
|US 3    | As a user, I should be able to upload attachments (such as documents or presentations) to support my funding application. |
|US 4    | As a user, I should be able to track the status of my funding applications and receive updates. |
|US 5    | As a fund manager, I should be able to advertise funding opportunities for educational, business, and event initiatives. |
|US 6    | As a fund manager, I should be able to review and approve/reject funding applications submitted by users. |
|US 7    | As a platform admin, I should be able to manage users, including reviewing and approving fund managers, blocking users, and managing permissions. |
|US 8    | As a user, I should be able to filter funding opportunities based on criteria such as funding type, deadline, and eligibility criteria. |
|US 9    | As a user, I should receive notifications or emails regarding the status of my funding application, such as when it has been reviewed or approved/rejected. |
|US 10   | As a fund manager, I should be able to set criteria and requirements for each funding opportunity, such as minimum results attained for educational scholarships or business plan requirements for business funding. |
|US 11   | As a platform admin, I should be able to generate and export reports on funding applications, including details such as applicant demographics, funding amounts requested, and funding types. |
|US 12   | As a user, I should be able to save draft versions of my funding applications and return to complete them later. |
|US 13   | As a fund manager, I should be able to allocate funds to specific funding opportunities and track the usage of allocated funds. |
|US 14   | As a platform admin, I should be able to customize the application's branding and appearance. |


## Sprint Backlogs

### Sprint 3 Backlog
- **Fund Manager and Application Management**:
  - Task 1: Implement functionality for fund managers to create and advertise funding opportunities (5 story points).
  - Task 2: Develop application review and approval/rejection process for fund managers (8 story points).
  - Task 3: Create functionality for applicants to submit applications with attachments (5 story points).
  - Task 4: Generate and export reports for admins on funds, applications, and system usage (8 story points).

## Sprint Retrospective Reports

### Sprint 3 Retrospective Report:

#### What Went Well:
- Everything is to expectation
- Successful implementation of the funding opportunities advertisement feature.
- Effective collaboration and communication within the team.
- Test cases and code coverage using jest fully working. 
- On-time completion of the application review and reporting functionalities.
- Team efficiently handled CI/CD pipeline improvements.
- Code cleaned up.

#### What Could Be Improved:
- Better management of task dependencies.
- More frequent check-ins with the client for early feedback on features.

#### Insights:
- Refinement of the database structure enhanced data retrieval performance.
- Integration of Firebase for authentication and SQL for data storage worked well.
- Effective division of tasks based on team member strengths ensured smooth progress.
- Continuous learning and adaptation to new tools and technologies.
- APIs added, Graphs added and now code created with testing in mind.

#### Action Items for Next Sprint:
- Enhance user interface for better user experience.
- Optimize backend processes for faster application review.
- Implement additional security measures for user data.
