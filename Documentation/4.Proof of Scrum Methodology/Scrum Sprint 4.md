## Product Backlog

**Product Backlog**:
### User Stories

| Identifier | User Story |
| ---------- | ------------- |
|US 1      | As a user, I should be able to register an account using a third-party identity provider, so that I can access the funding application platform securely. |
|US 2      | As a user, I should be able to log in using my third-party identity provider credentials, so that I can access my account. |
|US 3      | As a Fund Manager, I should be able to create and advertise funding opportunities, so that users can apply for funding. |
|US 4      | As a user, I should be able to apply for available funding opportunities, so that I can receive financial support for my needs. |
|US 5      | As a user, I should be able to upload and view attachments with my application, so that I can provide necessary documentation. |
|US 6      | As a user, I should be able to follow up on my applications and see status updates, so that I can stay informed about my application process. |
|US 7      | As a Fund Manager, I should be able to review and approve or reject applications, so that I can manage the funding opportunities effectively. |
|US 8      | As a Fund Manager, I should be able to view the total available budget and budget usage for my funding opportunities, so that I can effectively manage and allocate funds. |
|US 9      | As a Fund Manager, I should be able to allocate funds to approved applications, so that I can distribute the funds appropriately. |
|US 10      | As a Fund Manager, I should be able to generate reports on budget usage and allocation, so that I can track and analyze the budget effectively. |
|US 11      | As a Fund Manager, I want to be able to generate reports on application data, so that I can analyze and make informed decisions about funding opportunities. |
|US 12      | As a Fund Manager or Platform Admin, I want to be able to create custom reports based on specific criteria, so that I can analyze and report on specific aspects of the system. |
|US 13      | As a user, I should be able to receive notifications when the status of my application changes, so that I can stay informed about the progress of my application. |
|US 14      | As a Fund Manager, I should be able to receive notifications when new applications are submitted for my funding opportunities, so that I can review them in a timely manner. |

# Sprint Retrospective Reports

## Sprint 4 Retrospective Report:

**What Went Well:**
- The review and approval functionality for fund managers was successfully implemented and tested.
- The budgeting tools were completed and integrated without major issues.
- Report generation and export features were developed and worked as expected.
- The notification system was effectively set up, enhancing user experience.

**What Could Be Improved:**
- Some tasks took longer than initially estimated, highlighting the need for more accurate time estimation.
- Coordination between frontend and backend teams could be improved to avoid integration delays.
- Improve look of the app
- Test cases may need to be improved.
- Some glitches in code that was previously working. 

**Insights:**
- Choosing Firebase for authentication facilitated a smooth implementation of Google login.
- Using SQL for the main database operations proved to be efficient for managing complex queries.
- The decision to implement a notification system early on greatly improved user satisfaction and engagement.
- Time pressure of end of semester and burnout was evident although everything was added eventually. 

**Action Items for Next Sprint:**
- Enhance communication and coordination between frontend and backend teams.
- Refine time estimation practices based on the lessons learned from this sprint.
- Focus on user interface improvements and bug fixes to polish the final product.
